# Vercel + Supabase Setup

## 1) Create Supabase table
Run this SQL in your Supabase SQL Editor:

```sql
create table if not exists public.contact_messages (
  id bigint generated by default as identity primary key,
  name text not null,
  email text not null,
  subject text not null,
  message text not null,
  created_at timestamptz not null default now()
);
```

## 2) Add Vercel environment variables
In your Vercel project settings, add:

- `SUPABASE_URL` = your Supabase project URL (example: `https://xyzcompany.supabase.co`)
- `SUPABASE_SERVICE_ROLE_KEY` = your Supabase service role key
- `ADMIN_TOKEN` = any strong private string (used for `/admin/messages.html`)

After adding env vars, redeploy.

## 3) Form and admin URLs
- Contact form posts to: `/api/contact`
- Admin messages page: `/admin/messages.html`
  - It will ask for `ADMIN_TOKEN`
- Health check endpoint: `/api/health`
  - Returns `status: "ok"` when env vars are loaded
  - Returns `missing: [...]` if any required env var is not set
